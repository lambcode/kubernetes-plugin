By default (this option unchecked), the 'podTemplate' pipeline step can assign any service account to the new pod template. With this option checked, only defined service accounts can be used in dynamic templates. Note that predefined templates will not be affected by this option.

With this option turned on only the kubernetes default service account is allowed by default for 'podTemplate' templates. To specify additional allowed service accounts, the 'withAllowedServiceAccounts' step must be used from a non-sandboxed pipeline library.
